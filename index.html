赛事定位

这不只是竞赛，而是一场面向 MoonBit 开源生态的增长计划

MGPIC 2026 以 GitHub 开源协作为主要组织方式，以工程化交付为结果，以开发者长期留存为目标。 我们希望通过任务引导、项目孵化、过程激励和成果展示，沉淀一批真正能进入 MoonBit 生态的代表作。

开源优先

所有核心成果围绕公开仓库、提交历史、文档、演示与持续维护展开，鼓励真实协作而不是一次性冲奖。

工程导向

重点考察系统设计、实现质量、测试、可运行性与可演进性，让优秀项目具备长期价值。

生态沉淀

优秀项目将进入展示与孵化通道，反哺 MoonBit 社区内容、样例工程、工具链和包生态建设。

参与路径

从新人上手到主赛挑战，形成完整的开源漏斗

Path A

开源入门组

面向第一次接触 MoonBit 的开发者，完成环境搭建、Starter Task、首个 Issue / PR、最小可运行项目。

  • 官方任务池
  • 模板仓库与脚手架
  • 新手文档与示例
Path B

开源贡献组

围绕 MoonBit 生态项目、文档、测试、工具链和社区仓库展开持续贡献，重点看贡献质量与活跃度。

  • Good First Issue
  • 文档 / 测试 / 示例任务
  • 周度活跃与贡献榜
Path C

项目挑战组

面向希望做出完整项目和冲击大奖的开发者，以项目申报、里程碑开发、答辩评审为主线进行孵化。

  • 项目申报与预答辩
  • 持续提交与里程碑验收
  • Demo Day 与成果展示
重点机制

让 1000 人真正动起来的，不是报名页，而是承接系统

官方开源任务池

拆分新手、进阶、项目级任务，把围观者转成真实贡献者。

Starter Repo

提供模板仓库、License、README、脚手架和演示结构，缩短启动时间。

动态开源看板

实时展示参赛人数、仓库数、PR 数、commit 数、活跃高校与热门项目。

过程激励体系

奖励首次提交、首个 PR、连续活跃周、高质量文档和最佳新人贡献。

目标拆解

从一次活动，到一条生态增长漏斗

  1. 大规模触达技术社区与校园开发者
  2. 通过任务池、工作坊和模板仓库完成启动
  3. 用开源贡献和项目孵化筛选长期参与者
  4. 将优秀项目导入展示、孵化和长期维护通道

支持计划

Codex 启动支持 × 开源之夏联动 × 长期项目孵化

Codex 启动支持

为选手提供启动期 AI 编程支持，帮助完成仓库初始化、样例生成、任务拆解、文档搭建与开发提速。

  • 适合作为入门与主赛项目的启动层工具
  • 可与官方模板仓库、任务池和工作坊结合使用
  • 支持从“不会开始”到“可以开干”的过渡

与开源之夏式机制衔接

将 MGPIC 中表现突出的学生与项目，衔接到导师制、长期开发和正式项目孵化的路径中。

  • 前端是 1000 人规模的开源启动计划
  • 后端是项目制、导师制和中长期贡献机制
  • 形成“拉新 → 训练 → 孵化 → 沉淀”的完整闭环

FAQ

常见问题

这次赛事和往年最大的区别是什么？

2026 更强调开源协作、工程实践与长期项目沉淀，不只是一次性提交作品，而是围绕公开仓库与持续开发展开。

没有 MoonBit 经验也能参加吗？

可以。官网将提供 Starter Repo、任务池、训练营和入门路径，帮助新选手完成从 0 到 1 的启动。

Codex 支持会覆盖哪些阶段？

建议优先覆盖项目启动、任务拆解、文档搭建和样例开发阶段，再按项目表现分层提供更深入的支持。

是否会和开源之夏联动？

官网当前按照可联动方向预留了结构，后续可接入正式合作信息、项目制说明和导师机制。
