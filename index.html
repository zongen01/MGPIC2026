赛事定位

把一场赛事，做成一次真正的开源生态增长

MGPIC 2026 的重点不只是“交作品”，而是推动更多开发者围绕 MoonBit 建立公开仓库、持续提交代码、完成项目演进，并沉淀可展示、可复用、可继续维护的开源成果。

开源优先

围绕 GitHub 仓库、提交历史、README、演示与持续维护展开，鼓励真实协作与长期积累。

工程导向

重点关注系统设计、实现质量、可运行性、文档、测试与后续演进能力。

生态沉淀

优秀项目将进入展示、孵化与长期共建通道，反哺 MoonBit 样例、内容与工具生态。

参与路径

让不同阶段的开发者，都能顺利进入 MoonBit 开源生态

Path A

开源入门组

面向第一次接触 MoonBit 的开发者，通过 Starter Repo、示例任务与工作坊完成从 0 到 1 的启动。

  • 环境搭建与最小项目
  • 首个 Issue / PR
  • 新手任务池
Path B

开源贡献组

围绕 MoonBit 生态项目、文档、测试、工具链与社区仓库进行持续贡献，强调贡献质量与活跃度。

  • Good First Issue
  • 文档 / 测试 / 示例任务
  • 周度贡献榜
Path C

项目挑战组

面向希望做出完整项目和冲击重点展示的开发者，以项目申报、里程碑开发和最终答辩为主线推进。

  • 项目申报与预答辩
  • 中期验收与 Demo Day
  • 项目展示与孵化推荐

时间安排

建议采用五阶段推进赛事与开源增长

01

预热与合作招募

发布视觉与主张，启动社区、高校与合作伙伴联动。

02

报名与训练营

开放预登记，发布 Starter Repo、任务池与直播答疑。

03

开源贡献与项目开发

围绕公开仓库、持续提交与项目孵化展开正式开发。

04

中期验收与 Demo Day

通过活跃度、里程碑与项目质量筛选重点项目。

05

决赛答辩与成果开放

展示优秀项目，发布榜单，并开启后续生态共建通道。

支持计划

降低启动门槛，同时把优秀项目引向更长期的开源路径

Codex 启动支持

帮助选手完成仓库初始化、任务拆解、样例生成、文档搭建与开发提速，重点解决“不会开始”和“起步太慢”的问题。

开源任务池与模板仓库

通过 Starter Repo、Good First Issue 与任务池，让不同能力层级的开发者都有明确入口。

与开源之夏式项目孵化衔接

FAQ

常见问题

这次赛事和往年最大的区别是什么？

2026 更强调开源协作、工程实践与长期沉淀，不只是一次性提交作品，而是围绕公开仓库与持续开发展开。

没有 MoonBit 经验也能参加吗？

可以。官网结构里已经预留 Starter Repo、任务池和入门训练营入口，便于新人完成启动。

Codex 会在什么阶段发挥作用？

更适合项目启动、任务拆解、文档搭建、样例生成和开发提速阶段，后续可根据项目表现分层支持。

是否会和开源之夏形成联动？

页面已按照可联动方向预留结构，后续可继续补充正式合作信息、项目机制与导师安排。
